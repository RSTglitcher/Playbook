::tip Claude
**What it is:**
A long‑context, reasoning‑optimised AI assistant built by Anthropic.

- **Year developed:** 2023–2025
- **Primary purpose:** Deep reasoning, analysis, and structured synthesis
- **Target problem:** “I need to understand, transform, or reason about complex information.”
:::

---

## 🗺️ Context

### History
Claude was created by Anthropic with a focus on:

- safety
- reasoning
- long‑context understanding
- reliability
- clarity of expression

It evolved from:

- Claude 1 → early reasoning
- Claude 2 → long‑context breakthroughs
- Claude 3 → multimodal and high‑accuracy reasoning
- Claude 3.5 → state‑of‑the‑art structured thinking

Claude is widely used for:

- document analysis
- research
- rewriting
- system design
- knowledge work
- long‑form reasoning

### Strengths
- Exceptional long‑context understanding
- Strong at structured reasoning
- Excellent at rewriting and clarity
- Great at analysing documents, code, and systems
- High reliability and low hallucination rate
- Strong at step‑by‑step problem solving

### Limitations
- Not a code‑completion tool (use Copilot)
- Not repo‑aware (use Augment)
- Can be slower with extremely large inputs
- Less creative than ChatGPT in some domains

### TA‑Relevant Insights
Claude is uniquely powerful for Technical Analysts because it:

- synthesises complex requirements
- rewrites documents with precision
- analyses architecture and patterns
- identifies gaps, risks, and assumptions
- supports structured thinking and decision‑making
- handles long documents without losing context

Claude is your **analysis engine**.

## 🧠 Mental Model

### How to Think in Claude
Claude is not a code generator or autocomplete engine.
It is a **reasoning partner**.

Think of Claude as:

- a systems analyst
- a technical writer
- a requirements synthesiser
- a logic engine
- a clarity amplifier

Claude excels when you:

- give structure
- provide context
- ask for reasoning
- request transformations
- explore ideas step‑by‑step

### Ecosystem
Claude interacts with:

- long documents
- PDFs
- code snippets
- architectural diagrams
- research notes
- requirements
- meeting transcripts

It becomes part of your **thinking workflow**, not just your writing workflow.

## 🔁 Common Patterns

### Idioms & Typical Approaches
- “Summarise this…” → synthesis
- “Rewrite this…” → clarity
- “Analyse this…” → reasoning
- “Extract requirements…” → TA workflows
- “Identify risks…” → design reviews
- “Compare options…” → decision support
- “Trace the flow…” → system understanding

### Pitfalls
- Giving too little context
- Asking for code generation better suited to Copilot
- Using Claude for repo‑wide reasoning (use Augment)
- Overloading with unstructured data
- Forgetting to specify format or structure
