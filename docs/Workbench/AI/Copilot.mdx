## 🧩 Medium Definition

:::tip Microsoft Copilot
**What it is:**
A cross‑application AI companion built into Microsoft’s ecosystem (Windows, Edge, Office, Teams, and the web).

- **Year developed:** 2023–2025
- **Primary purpose:** General reasoning, productivity, and workflow acceleration
- **Target problem:** “Help me think, plan, write, analyse, and act across my tools.”
:::

---

## 🗺️ Context

### History
Microsoft Copilot emerged from the convergence of:

- LLMs
- Microsoft Graph
- Office productivity tools
- Windows system integration
- Edge browser workflows

It evolved from:

- Bing Chat →
- Copilot in Windows →
- Copilot in Office →
- Copilot across the Microsoft ecosystem

It is now a **system‑level assistant** that spans:

- documents
- spreadsheets
- presentations
- emails
- meetings
- browsing
- planning
- reasoning
- summarisation

### Strengths
- Integrated across your entire workflow
- Strong at summarising and rewriting
- Excellent at planning and structuring tasks
- Great at cross‑document synthesis
- Can act on your behalf inside Microsoft apps
- Strong multimodal reasoning (text + images)
- Always available, context‑aware

### Limitations
- Not repo‑aware (use Augment)
- Not code‑completion focused (use GitHub Copilot)
- Not as long‑context as Claude
- Dependent on app integration for actions
- Requires clear instructions for multi‑step tasks

### TA‑Relevant Insights
Microsoft Copilot is uniquely powerful for Technical Analysts because it:

- synthesises meeting notes, documents, and requirements
- rewrites content with clarity and structure
- analyses spreadsheets and data tables
- drafts technical documents, specs, and summaries
- supports planning and decision‑making
- integrates with your existing tools (Word, Excel, Teams)
- helps you move between thinking, writing, and action seamlessly

It is your **generalist thinking partner**.

## 🧠 Mental Model

### How to Think in Microsoft Copilot
Microsoft Copilot is not a coding assistant or a repo interpreter.
It is a **workflow‑level reasoning environment**.

Think of it as:

- a planning engine
- a writing companion
- a summarisation tool
- a cross‑document analyst
- a productivity orchestrator
- a cognitive amplifier

It excels when you:

- give it context
- ask for structure
- request transformations
- use it across apps
- treat it as a collaborator

### Ecosystem
Microsoft Copilot interacts with:

- Word (drafting, rewriting, summarising)
- Excel (analysis, formulas, insights)
- PowerPoint (slide generation, structure)
- Outlook (email drafting, summarisation)
- Teams (meeting summaries, action items)
- Edge (web analysis, research)
- Windows (system‑level tasks)

It becomes part of your **daily workflow**, not a separate tool.

## 🔁 Common Patterns

### Idioms & Typical Approaches
- “Summarise this…” → synthesis
- “Rewrite this…” → clarity
- “Extract…” → requirements
- “Plan…” → structure
- “Compare…” → decision support
- “Analyse…” → reasoning
- “Generate slides…” → communication

### Pitfalls
- Giving too little context
- Expecting repo‑level understanding (use Augment)
- Using it for code completion (use GitHub Copilot)
- Overloading with unstructured data
- Forgetting to specify output format
