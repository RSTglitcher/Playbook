:::tip Embeddings
**What they are:**
High‑dimensional vectors that represent the meaning of text, images, or data.

- **Year developed:** 2013–present
- **Primary purpose:** Encode semantic meaning in a machine‑interpretable form
- **Target problem:** “How can a system understand similarity, relationships, and meaning?”
:::

---

## 🗺️ Context

### History
Embeddings emerged from early NLP research:

- Word2Vec (2013) → word embeddings
- GloVe (2014) → global co‑occurrence
- BERT embeddings (2018) → contextual meaning
- Modern LLM embeddings (2020s) → sentence, document, multimodal embeddings

They now power:

- semantic search
- RAG
- recommendation systems
- clustering
- classification
- anomaly detection
- personalisation

### Strengths
- Capture semantic meaning
- Enable similarity search
- Work across languages and domains
- Efficient to store and compare
- Foundation of RAG and vector search

### Limitations
- Dependent on model quality
- Sensitive to domain mismatch
- Hard to interpret directly
- Require vector stores for scale
- Not suitable for exact matching

### TA‑Relevant Insights
Embeddings matter for Technical Analysts because they:

- explain why RAG retrieves certain documents
- reveal how AI systems “understand” meaning
- help diagnose retrieval failures
- underpin personalisation and recommendation logic
- influence ranking, clustering, and similarity scoring
- determine how well AI systems connect concepts

Embeddings are the **semantic substrate** of AI systems.

---

## 🧠 Mental Model

### How to Think in Embeddings
Embeddings are not code or prompts.
They are **coordinates in meaning‑space**.

Think of embeddings as:

- a map of concepts
- where similar ideas cluster together
- and unrelated ideas drift apart

Examples:

- “cat” is close to “kitten”
- “invoice” is close to “billing”
- “authentication” is close to “login”
- “refund” is far from “authentication”

Embeddings let AI systems reason about:

- similarity
- relevance
- relationships
- clustering
- semantic distance

### Ecosystem
Embeddings interact with:

- vector stores
- RAG pipelines
- search engines
- recommendation systems
- clustering algorithms
- classification models

They are the **connective tissue** of AI systems.

Embedding Workflows

# Generate an embedding for text

# Store in vector DB
INSERT INTO vector_store (id, vector, metadata)

# Query by similarity
SELECT * FROM vector_store
ORDER BY vector <-> query_embedding
LIMIT 5;

## 🔁 Common Patterns

### Idioms & Typical Approaches
- “Find similar items…” → semantic search
- “Cluster these…” → grouping
- “Rank by relevance…” → retrieval
- “Match user to content…” → personalisation
- “Retrieve context for LLM…” → RAG
- “Detect anomalies…” → outlier detection

### Pitfalls
- Using embeddings for exact matching
- Mixing embeddings from different models
- Using embeddings without normalisation
- Ignoring domain mismatch
- Assuming embeddings are interpretable
- Forgetting that embeddings drift over time

## 📝 Glossary

| Term | Meaning |
|------|---------|
| **Vector** | A list of numbers representing meaning |
| **Embedding** | A vector that encodes semantic information |
| **Cosine Similarity** | A measure of how similar two vectors are |
| **Vector Store** | A database for storing and searching embeddings |
| **Semantic Search** | Search based on meaning, not keywords |
| **RAG** | Retrieval‑Augmented Generation, powered by embeddings |
