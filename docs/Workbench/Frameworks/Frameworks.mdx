Frameworks are **composable system structures** — you often use several together.

They provide:

- architecture
- conventions
- abstractions
- workflows
- integration points

Frameworks are **building blocks**, not exclusive environments.
They layer, combine, and interact.

### What belongs here
- Spring
- Kafka
- React
- Express
- Terraform
- AWS

### How to use this directory
Use these pages to understand:

- how frameworks structure systems
- how they interact with each other
- common patterns and pitfalls
- how engineers use them in practice
- how analysts reason about their behaviour

Frameworks define **how systems are built**.
