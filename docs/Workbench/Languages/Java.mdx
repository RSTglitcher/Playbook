:::tip Java
Java is a statically‑typed, object‑oriented programming language designed for portability, reliability, and large‑scale systems.

- **Year developed:** 1995
- **Primary purpose:** Enterprise applications, backend services, large distributed systems
- **Target problem:** Building robust, maintainable, cross‑platform software
:::

---

## 🗺️ Context

### History
Java was created by **James Gosling** at Sun Microsystems with the philosophy:

> “Write once, run anywhere.”

Its JVM‑based architecture made it the dominant language for:

- Enterprise systems
- Banking & finance
- Android development
- Distributed backend services
- High‑reliability systems

### Strengths & Limitations

**Strengths**
- Strong static typing
- Mature ecosystem and tooling
- JVM performance and stability
- Huge enterprise adoption
- Excellent for long‑lived systems

**Limitations**
- Verbose syntax
- Slower iteration compared to dynamic languages
- Requires more boilerplate
- Functional features arrived later (Java 8+)

### TA‑Relevant Insights
- Java codebases often reflect **enterprise architecture patterns**
- Understanding Java helps you interpret Spring Boot services, microservices, and backend APIs
- Many large organisations still rely heavily on Java for mission‑critical systems
- Java’s type system reveals domain boundaries and invariants clearly

---

## 🧠 Mental Model

### How to think in this environment
- Java is **class‑centric** — everything lives inside a class
- Types define **contracts**, **constraints**, and **behaviour**
- Execution is **imperative** and **object‑oriented**
- The JVM abstracts away hardware differences
- Concurrency is explicit (threads, executors, futures)

### Ecosystem
- **Build tools:** Maven, Gradle
- **Frameworks:** Spring Boot, Jakarta EE, Micronaut
- **Data:** JDBC, JPA, Hibernate
- **Android:** Kotlin + Java interop
- **Tooling:** IntelliJ, Eclipse, VS Code
- **JVM:** HotSpot, OpenJDK

## Conditionals

Java uses **curly braces `{}`** to define code blocks.
Conditionals allow your program to **make decisions** — a core interview topic.

---

### IF Statements

#### 1‑Way IF
**Why used:** To run logic only when a condition is true.
**Example:** *Check if a shopping basket is empty before checkout.*

```java
if (basket.isEmpty()) {
    System.out.println("Your basket is empty.");
}
```

---

#### 2‑Way IF / ELSE
**Why used:** To choose between two mutually exclusive outcomes.
**Example:** *Apply free shipping if order total exceeds £50.*

```java
if (orderTotal >= 50) {
    System.out.println("Free shipping applied.");
} else {
    System.out.println("Shipping cost: £4.99");
}
```

---

#### Multi‑Way IF / ELSE IF / ELSE
**Why used:** To handle multiple possible states.
**Example:** *Categorise user login attempts.*

```java
if (attempts == 0) {
    System.out.println("First attempt");
} else if (attempts < 3) {
    System.out.println("Retry allowed");
} else {
    System.out.println("Account locked");
}
```

---

### Loops

#### FOR Loop (Index‑Based)
**Why used:** When you need the index or want precise control.
**Example:** *Iterate through product IDs by index.*

```java
for (int i = 0; i < productIds.size(); i++) {
    System.out.println(productIds.get(i));
}
```

---

#### FOR‑EACH Loop
**Why used:** When you want clean iteration without managing indexes.
**Example:** *Print all items in a shopping basket.*

```java
for (String item : basket) {
    System.out.println(item);
}
```

---

#### FOR‑EACH with HashMap
**Why used:** To iterate through key/value pairs.
**Example:** *Display product → quantity in a basket.*

```java
for (Map.Entry<String, Integer> entry : basket.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}
```

---

#### IF + FOR Combined
**Why used:** To apply conditional logic to each element.
**Example:** *Identify discounted products in a list.*

```java
for (Product p : products) {
    if (p.isDiscounted()) {
        applyDiscount(p);
    }
}
```

---

### WHILE Loop
**Why used:** When you loop until a condition changes.
**Example:** *Retry API call until success or timeout.*

```java
int retries = 0;

while (!apiCallSuccess && retries < 3) {
    apiCallSuccess = callApi();
    retries++;
}
```

---

### DO‑WHILE Loop
**Why used:** When the logic must run at least once.
**Example:** *Prompt user for input until valid.*

```java
String input;

do {
    input = scanner.nextLine();
} while (!isValid(input));
```

## Sorting

Sorting is a common interview topic.
These interfaces define **how objects should be ordered**.

---

### Comparable
**Why used:** To define a class’s *natural ordering* (e.g., alphabetical).
**Example:** *Sort books alphabetically by title.*

```java
public class Book implements Comparable<Book> {
    private String title;

    @Override
    public int compareTo(Book other) {
        return this.title.compareTo(other.title);
    }
}
```

Usage:
```java
Collections.sort(bookList);
```

---

### Comparator
**Why used:** To define *custom* or *multiple* sorting strategies.
**Example:** *Sort products by price.*

```java
public class ProductByPrice implements Comparator<Product> {
    @Override
    public int compare(Product p1, Product p2) {
        return Double.compare(p1.getPrice(), p2.getPrice());
    }
}
```

Usage:
```java
Collections.sort(products, new ProductByPrice());
```

## Generics

**Why used:** To make collections type‑safe and prevent runtime casting errors.
**Example:** *Ensure a list only contains Product objects.*

---

### Generic Class Example
```java
public class Box<T> {
    private T value;

    public Box(T value) {
        this.value = value;
    }

    public T getValue() {
        return value;
    }
}
```

Usage:
```java
Box<String> stringBox = new Box<>("Hello");
Box<Integer> intBox = new Box<>(42);
```

---

### Bounded Generics (`extends`)
**Why used:** To restrict a generic type to a specific hierarchy.
**Example:** *Allow only numeric types for a statistics calculator.*

```java
public class NumberBox<T extends Number> {
    private T number;

    public NumberBox(T number) {
        this.number = number;
    }
}
```

Usage:
```java
NumberBox<Integer> box = new NumberBox<>(10);
```

## Optionals

**Why used:** To avoid `NullPointerException` and represent “value may or may not exist.”
**Example:** *Handle missing user email.*

---

### Creating Optionals
```java
Optional<String> email = Optional.of("user@example.com");
Optional<String> empty = Optional.empty();
```

---

### Using Optionals
```java
email.ifPresent(e -> System.out.println("Email: " + e));

String result = email.orElse("No email provided");
```

---

### Avoiding Null Checks
```java
String value = optionalValue
        .map(String::toUpperCase)
        .orElse("DEFAULT");
```

## File I/O

File I/O is used to **read and write data**, common in backend systems.

---

### Reading a File (BufferedReader)
**Why used:** Efficient for reading large files line‑by‑line.
**Example:** *Process a CSV of product data.*

```java
try (BufferedReader reader = new BufferedReader(new FileReader("products.csv"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

---

### Reading a File (Scanner)
**Why used:** Simple API for reading tokens or lines.
**Example:** *Read configuration values from a text file.*

```java
try (Scanner scanner = new Scanner(new File("config.txt"))) {
    while (scanner.hasNextLine()) {
        System.out.println(scanner.nextLine());
    }
} catch (FileNotFoundException e) {
    e.printStackTrace();
}
```

---

### Writing to a File
**Why used:** To persist data or generate output files.
**Example:** *Export order summary to a file.*

```java
try (FileWriter writer = new FileWriter("order-summary.txt")) {
    writer.write("Order Total: £120.00\n");
} catch (IOException e) {
    e.printStackTrace();
}
```

## Streams API

**Why used:** Streams allow functional‑style operations on collections.
**Example:** *Filter products above a certain price.*

---

### Basic Stream Example
```java
List<String> names = List.of("Alice", "Bob", "Charlie");

names.stream()
     .filter(n -> n.startsWith("A"))
     .map(String::toUpperCase)
     .forEach(System.out::println);
```

---

### Stream Reduce
**Why used:** To combine elements into a single result.
**Example:** *Calculate total basket quantity.*

```java
int total = quantities.stream()
                      .reduce(0, Integer::sum);
```

---

### Stream Collect
**Why used:** To transform a stream back into a collection.
**Example:** *Convert product names to uppercase list.*

```java
List<String> upper =
    names.stream()
         .map(String::toUpperCase)
         .collect(Collectors.toList());
```
