:::tip REST APIs
REST APIs provide a standardized way for systems to communicate over HTTP using resource‑based endpoints. They enable applications to exchange data reliably, integrate services, and expose functionality in a scalable, language‑agnostic manner.
:::

<TabItem value="requirements" label="Requirements">

**How to formulate requirements when working with this platform**

- Clarify the purpose of each endpoint and the business workflow it supports
- Identify required inputs, optional parameters, and validation rules
- Confirm authentication method and environment differences
- Understand expected response formats and error handling
- Capture edge cases, rate limits, and performance expectations

**Examples of use cases**

- Integrating two internal systems
- Supporting frontend development with backend endpoints
- Validating data returned from external APIs
- Documenting API behaviour for cross‑team alignment

**What a TA needs to gather**

- API specification (OpenAPI/Swagger/Postman collection)
- Authentication details and environment URLs
- Request/response examples
- Data mapping rules and transformation logic
- Error scenarios and fallback expectations

- Identify validation rules and error handling patterns
- Plan integration with downstream systems

**Examples of implementation**

- CRUD endpoints for a new feature
- API gateway routing and aggregation
- Middleware for logging, auth, or rate limiting
- Automated contract tests

**Engineering considerations**

- Idempotency for PUT/PATCH
- Pagination for large datasets
- Consistent error formats

**What a TA needs to provide for handover**

## Triage

:::danger
Symptoms and issues with this platform and how to triage this:
:::

| Symptom | TLDR of Root Cause | What to Check | Action to Propose |
|---------|---------------------|---------------|-------------------|
| 400 Bad Request | Invalid input or missing fields | Request body, params, validation rules | Fix payload or update validation |
| 401/403 Errors | Auth failure or missing permissions | Tokens, headers, roles | Refresh token, update permissions |
| 404 Not Found | Wrong endpoint or version | URL, path params, API version | Correct route or update spec |
| 429 Too Many Requests | Rate limit exceeded | Rate limit headers | Add retries/backoff |
| 500 Errors | Server-side failure | Logs, downstream dependencies | Escalate to engineering, check integrations |
