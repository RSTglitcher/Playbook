:::tip Git
Git is a **distributed version control system** used to track changes in source code, collaborate across teams, manage branches, and coordinate releases.
It solves problems related to **source code history, collaboration, branching strategies, and reproducible deployments**.
:::

### Key Terms

- **Repository:** A project tracked by Git
- **Commit:** A snapshot of changes
- **Branch:** A movable pointer to a commit history
- **Merge:** Combine changes from one branch into another
- **Rebase:** Replay commits on top of another branch
- **Tag:** A named reference to a specific commit (often releases)
- **Remote:** A reference to a hosted repository (GitHub, GitLab, Azure DevOps)
- **HEAD:** The currently checked‑out commit or branch

### Knowing when to use **Git**

| When to use | When not to use |
|-------------|-----------------|
| Source code versioning | Binary file storage (use artifact storage) |
| Collaboration via branches and PRs | Real‑time document editing |
| Release management and tagging | Large dataset versioning (use DVC/LakeFS) |
| CI/CD integration | Database schema versioning (use migrations) |

## 📘 About / Foundations
_Conceptual grounding for how the tool works._

### How It Works (Conceptual Model)

- Git tracks changes using **snapshots**, not diffs
- Commits form a **directed acyclic graph (DAG)**
- Branches are lightweight pointers to commits
- Remotes allow distributed collaboration
- Merges and rebases integrate changes across branches
- Tags mark important commits (e.g., releases)
- `.git` directory stores all repository metadata

### System Contract

Git enforces contracts through:

- **Repository structure** (`.git` directory)
- **Branching model** (feature branches, trunk‑based, GitFlow)
- **Commit history** (immutable snapshots)
- **Remote configuration** (origin, upstream)
- **Authentication** (SSH keys, tokens)
- **Protected branches** (main, release branches)

### Key Strategies

<Tabs>
<TabItem value="branching" label="Branching Strategy">

- Use feature branches for isolated work
- Keep `main` stable and deployable
- Use PRs for code review and quality gates
- Choose between GitFlow or trunk‑based depending on team maturity
- Responsibility: SWE + TA validation
- Goal: predictable, conflict‑free collaboration

</TabItem>

<TabItem value="history" label="History Strategy">

- Prefer small, meaningful commits
- Use rebase for clean history
- Use merge for preserving context
- Responsibility: SWE
- Goal: readable, auditable commit history

</TabItem>

<TabItem value="release" label="Release Strategy">

- Use annotated tags for releases
- Automate release notes via CI/CD
- Protect release branches
- Responsibility: SWE + TA
- Goal: reproducible, traceable releases

</TabItem>
</Tabs>

### What Results Normally Look Like

- Commit logs (`git log --oneline --graph`)
- Branch lists (`git branch -a`)
- Diffs (`git diff`)
- PRs and merge histories
- Tags for releases
- CI/CD pipelines triggered by Git events

## 🚨 Triage

:::danger
Symptoms and issues with this platform and how to triage this:
:::

| Symptom | TLDR of Root Cause | What to Check | Action to Propose |
|--------|---------------------|---------------|-------------------|
| Cannot clone repository | Wrong URL, network issue, permission denied | Repo URL, network, SSH keys | Fix URL, check network, update keys |
| Merge conflicts | Concurrent edits to same files | `git status`, diff | Resolve conflicts manually |
| Push rejected | Protected branch or missing commits | Branch policy, `git log` | Pull latest, rebase or merge |
| Detached HEAD | Checked out commit instead of branch | `git status` | Checkout correct branch |
| Missing files | Ignored or untracked files | `.gitignore`, `git status` | Add files or update ignore rules |
| Commit not visible remotely | Local commits not pushed | `git log`, `git status` | Push branch |
| Authentication fails | Expired SSH key or token | `ssh -T git@host` | Rotate key or token |
| Slow operations | Large repo or binary files | Repo size, LFS usage | Use shallow clone, enable Git LFS |
