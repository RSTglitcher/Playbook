:::tip Google Cloud Platform (GCP)  
GCP is a **cloud platform** providing compute, storage, analytics, and messaging services.  
This runbook focuses on key data and messaging services used by engineers and monitored by TAs: **BigQuery, BigTable, Cloud Storage, and Pub/Sub**.  
:::

### Key Terms

- **Project:** Logical boundary for resources, billing, IAM  
- **Service Account:** Identity used by applications to access GCP services  
- **BigQuery Dataset:** Logical container for tables  
- **BigTable Instance:** Clustered NoSQL wide‑column store  
- **Pub/Sub Topic:** Channel for publishing messages  
- **Subscription:** Consumer endpoint for Pub/Sub messages  
- **Cloud Storage Bucket:** Object storage container  
- **IAM Role:** Permission set assigned to users or service accounts  

### Knowing when to use **GCP**

| When to use | When not to use |
|-------------|-----------------|
| Large‑scale analytics (BigQuery) | OLTP workloads requiring transactions |
| High‑throughput messaging (Pub/Sub) | Complex routing or ordering guarantees |
| Wide‑column NoSQL workloads (BigTable) | Small datasets or relational joins |
| Object storage for logs, files, ML artifacts | Low‑latency block storage |
| Serverless, autoscaling data pipelines | Real‑time compute (use Databricks/Spark) |

## 📘 About / Foundations  
_Conceptual grounding for how the tool works._

### How It Works (Conceptual Model)

- GCP resources live inside **projects**, governed by IAM  
- **BigQuery** is a serverless analytics warehouse  
- **BigTable** is a horizontally scalable NoSQL database  
- **Pub/Sub** is a global messaging system for event ingestion  
- **Cloud Storage** provides durable object storage  
- Services integrate through IAM, VPC networking, and APIs  
- Monitoring and logging are provided via **Cloud Monitoring** and **Cloud Logging**  

### System Contract

GCP enforces contracts through:

- **IAM roles and policies**  
- **Project and folder hierarchy**  
- **Service APIs** (must be enabled per project)  
- **Resource‑specific schemas** (BigQuery tables, BigTable column families)  
- **Pub/Sub retention and delivery settings**  
- **Cloud Storage bucket policies**  

### Key Strategies

<Tabs>
<TabItem value="bigquery" label="BigQuery Strategy">

- Use partitioning and clustering for large tables  
- Apply table expiration policies for cost control  
- Use IAM at dataset level for access boundaries  
- Responsibility: SWE + TA  
- Goal: scalable, cost‑efficient analytics  

</TabItem>

<TabItem value="pubsub" label="Pub/Sub Strategy">

- Use pull subscriptions for worker pools  
- Configure DLQs for failure handling  
- Tune ack deadlines and retry policies  
- Responsibility: SWE + TA  
- Goal: reliable, high‑throughput messaging  

</TabItem>

<TabItem value="bigtable" label="BigTable Strategy">

- Design row keys to avoid hotspots  
- Scale nodes based on throughput  
- Use replication for availability  
- Responsibility: SWE + Platform  
- Goal: predictable performance and durability  

</TabItem>
</Tabs>

- Pub/Sub backlog metrics and subscription details  
- BigTable table metadata and cluster performance  
- Cloud Storage bucket listings and object metadata  
- IAM policy bindings

## 🚨 Triage

:::danger  
Symptoms and issues with this platform and how to triage this:  
:::

| Symptom | TLDR of Root Cause | What to Check | Action to Propose |
|--------|---------------------|---------------|-------------------|
| BigQuery query fails | SQL error or insufficient permissions | Query syntax, IAM roles | Fix SQL, adjust IAM |
| Pub/Sub message loss | Misconfigured subscription or consumer down | Subscription config, consumer logs | Fix subscription, restart consumer |
| BigTable slow reads | Hotspot or insufficient nodes | Row key design, cluster size | Redesign keys, scale cluster |
| IAM access denied | Missing roles or policies | IAM bindings | Grant required roles |
| API quota exceeded | Too many requests | Quotas, logs | Request increase, throttle |
| Cloud Storage access denied | Bucket policy or missing permissions | IAM + bucket ACLs | Fix IAM, update bucket policy |
