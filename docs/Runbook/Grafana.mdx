:::tip Grafana  
Grafana is a **visualization and observability platform** used to explore metrics, logs, and traces from multiple data sources.  
It solves problems related to **monitoring, alerting, dashboarding, and operational insights** across distributed systems.  
:::

### Key Terms

- **Dashboard:** A collection of visualizations  
- **Panel:** A single chart, table, or visualization  
- **Data Source:** Backend providing metrics/logs (Prometheus, Loki, Elasticsearch, CloudWatch, etc.)  
- **Alert Rule:** Condition that triggers notifications  
- **Annotation:** Marker on a graph for events or deployments  
- **Folder:** Logical grouping of dashboards  
- **Variables:** Dynamic parameters used in dashboards  

### Knowing when to use **Grafana**

| When to use | When not to use |
|-------------|-----------------|
| Metrics visualization and dashboards | Heavy log search (use Kibana/Loki) |
| Alerting on metrics or thresholds | Complex analytics (use BigQuery/Snowflake) |
| Multi‑source observability | Real‑time event processing |
| SRE/TA operational monitoring | Business intelligence dashboards (use Looker/PowerBI) |
| Time‑series exploration | Schema‑driven reporting |

## 📘 About / Foundations  
_Conceptual grounding for how the tool works._

### How It Works (Conceptual Model)

- Grafana connects to **data sources** (Prometheus, Loki, CloudWatch, Elasticsearch, InfluxDB, etc.)  
- Dashboards are composed of **panels** that query data sources  
- **Alert rules** evaluate queries on a schedule  
- **Annotations** overlay events on charts  
- **Variables** allow dynamic filtering  
- **Folders** organize dashboards by team or domain  
- Grafana does **not** store metrics — it visualizes them  

### System Contract

Grafana enforces contracts through:

- **Data source configuration** (URLs, credentials, query language)  
- **Dashboard JSON models**  
- **Folder and dashboard permissions**  
- **Alerting rules and notification channels**  
- **Provisioning files** (optional, for GitOps)  

### Key Strategies

<Tabs>
<TabItem value="dashboards" label="Dashboard Strategy">

- Use variables for reusable dashboards  
- Group dashboards by domain (infra, application, messaging, storage)  
- Use consistent naming and tagging  
- Responsibility: SWE + TA  
- Goal: clear, actionable operational visibility  

</TabItem>

<TabItem value="alerting" label="Alerting Strategy">

- Use multi‑condition alerts (threshold + absence)  
- Route alerts by severity (Slack, PagerDuty, email)  
- Use silence windows for deployments  
- Responsibility: TA + Platform  
- Goal: reduce noise, increase signal  

</TabItem>

<TabItem value="data" label="Data Strategy">

- Choose the right data source for the right workload  
- Use Prometheus for metrics, Loki for logs, Elasticsearch for search  
- Ensure retention and scrape intervals match SLOs  
- Responsibility: SWE + TA  
- Goal: reliable, performant observability  

</TabItem>
</Tabs>

### What Results Normally Look Like

- Time‑series charts (CPU, memory, latency, throughput)  
- Heatmaps, histograms, and tables  
- Alert notifications  
- Dashboard JSON exports  
- Query inspector output  
- Variable‑driven dashboards

How to log-in via the terminal

## 🚨 Triage

:::danger  
Symptoms and issues with this platform and how to triage this:  
:::

| Symptom | TLDR of Root Cause | What to Check | Action to Propose |
|--------|---------------------|---------------|-------------------|
| Dashboard slow to load | Heavy queries, large time range | Query inspector, panel queries | Reduce time range, optimize queries |
| No data in panel | Data source misconfig or query error | Data source status, query inspector | Fix query, update data source |
| Alerts not firing | Wrong condition or data source down | Alert rule logs, data source health | Fix rule, restore data source |
| High query latency | Prometheus overload, Loki slow | Prometheus metrics, Loki logs | Scale Prometheus, optimize labels |
| Missing dashboards | Folder permissions or deleted objects | Folder ACLs, dashboard JSON | Restore from backup, fix permissions |
| Grafana not loading | Server down or network issue | Server logs, network path | Restart Grafana, fix networking |
