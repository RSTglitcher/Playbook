:::tip Kubernetes  
Kubernetes (K8s) is a **container orchestration platform** that automates deployment, scaling, and management of containerized applications.  
It enables microservices architectures, declarative infrastructure, and resilient distributed systems.  
:::

### Key Terms

- **Cluster:** A set of nodes running Kubernetes  
- **Node:** A machine (VM or physical) that runs workloads  
- **Pod:** The smallest deployable unit (one or more containers)  
- **Deployment:** Manages stateless workloads  
- **StatefulSet:** Manages stateful workloads  
- **Service:** Stable networking endpoint for pods  
- **Ingress:** External routing into the cluster  
- **ConfigMap / Secret:** Configuration and sensitive data  
- **HPA:** Horizontal Pod Autoscaler  
- **Namespace:** Logical grouping of resources  
- **Kubelet:** Node agent that runs pods  
- **Control Plane:** API server, scheduler, controller manager  

### Knowing when to use **Kubernetes**

| When to use | When not to use |
|-------------|-----------------|
| Microservices architectures | Simple single‑service apps |
| Autoscaling workloads | Batch jobs without orchestration needs |
| Multi‑team shared infrastructure | Small teams without ops capacity |
| Declarative infrastructure | Highly stateful monoliths |
| CI/CD‑driven deployments | Apps requiring bare‑metal performance |

## 📘 About / Foundations  
_Conceptual grounding for how the platform works._

### How It Works (Conceptual Model)

- Kubernetes manages workloads using **desired state**  
- The **control plane** reconciles actual vs desired state  
- **Pods** run containers; **Deployments** manage pods  
- **Services** provide stable networking; **Ingress** exposes apps externally  
- **ConfigMaps/Secrets** inject configuration  
- **HPA** scales workloads based on metrics  
- **Nodes** provide compute capacity  
- **Events** surface cluster activity and issues  

### System Contract

Kubernetes enforces contracts through:

- **YAML manifests** (Deployments, Services, Ingress, etc.)  
- **RBAC** (roles, service accounts, permissions)  
- **Resource requests/limits**  
- **Probes** (liveness, readiness, startup)  
- **Networking model** (CNI plugin)  
- **Storage classes and PV/PVC binding**  

### Key Strategies

<Tabs>
<TabItem value="workloads" label="Workload Strategy">

- Use Deployments for stateless apps  
- Use StatefulSets for stateful workloads  
- Use ConfigMaps/Secrets for configuration  
- Responsibility: SWE + TA  
- Goal: predictable, resilient workloads  

</TabItem>

<TabItem value="scaling" label="Scaling Strategy">

- Use HPA for CPU/memory‑based scaling  
- Use VPA/Cluster Autoscaler for node scaling  
- Monitor pod restarts and resource pressure  
- Responsibility: Platform + SWE  
- Goal: efficient, cost‑aware scaling  

</TabItem>

<TabItem value="networking" label="Networking Strategy">

- Use Services for internal routing  
- Use Ingress for external routing  
- Validate network policies for security  
- Responsibility: SWE + TA  
- Goal: secure, reliable connectivity  

</TabItem>
</Tabs>

<TabItem value="engineering" label="Engineering">

Common workflows and commands used by engineers when operating this platform

```shell
--------------
Workload Management
--------------

# Deploy workloads
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml

# Inspect pods
kubectl get pods -n NAMESPACE
kubectl describe pod POD_NAME -n NAMESPACE

# Debug workloads
kubectl logs POD_NAME -n NAMESPACE
kubectl exec -it POD_NAME -- sh

# Scale workloads
kubectl scale deployment/<name> --replicas=3

# Manage rollouts
kubectl rollout status deployment/<name>
kubectl rollout undo deployment/<name>

# Node health
kubectl get nodes
kubectl describe node NODE_NAME
kubectl top nodes

# Storage
kubectl get pv
kubectl get pvc -n NAMESPACE

# Networking
kubectl get svc -n NAMESPACE
kubectl get ingress -n NAMESPACE
kubectl get endpoints SERVICE_NAME -n NAMESPACE
```

## 🚨 Triage

:::danger  
Symptoms and issues with Kubernetes clusters and how to triage:  
:::

| Symptom | TLDR of Root Cause | What to Check | Action to Propose |
|---------|---------------------|---------------|-------------------|
| Pod CrashLoopBackOff | Container crash or misconfig | `describe pod`, logs | Fix image/config, restart pod |
| Node Not Ready | Node down or network issue | `get nodes`, node events | Restart node/kubelet |
| Deployment Not Scaling | HPA misconfig or missing metrics | `describe hpa` | Adjust HPA or resource limits |
| PersistentVolume Pending | Storage class or PV mismatch | `get pv,pvc`, events | Provision storage or fix binding |
| Service Unreachable | Networking or service misconfig | svc, endpoints, network policies | Fix service/ingress config |
| API Server Unresponsive | Control plane overload | API server logs | Scale control plane, validate RBAC |
| ConfigMap/Secret Missing | Resource deleted or misconfigured | `get configmap/secret` | Restore or recreate resource |
