:::tip Databricks
Databricks is a **unified analytics and data engineering platform** built on top of Apache Spark.
It provides collaborative notebooks, scalable compute clusters, Delta Lake storage, and orchestration for ETL, ML, and analytics workloads.
It solves problems related to **large‑scale data processing, lakehouse architecture, streaming, and collaborative development**.
:::

### Key Terms

- **Workspace:** The Databricks environment containing notebooks, repos, jobs, clusters
- **Cluster:** Compute resources used to run notebooks, jobs, and queries
- **DBFS:** Databricks File System (abstraction over cloud storage)
- **Delta Lake:** ACID‑compliant storage layer for data lakes
- **Job:** Scheduled or triggered workload (ETL, ML training, batch processing)
- **Notebook:** Interactive development environment for SQL, Python, Scala, R
- **Unity Catalog:** Centralised governance for data access and lineage

### Knowing when to use **Databricks**

| When to use | When not to use |
|-------------|-----------------|
| Large‑scale ETL/ELT workloads | Small datasets easily handled by a warehouse |
| Collaborative notebooks for data science | Real‑time low‑latency APIs |
| Lakehouse architecture (Delta Lake) | Traditional OLTP workloads |
| ML training and feature engineering | Simple SQL transformations (use dbt) |
| Streaming pipelines (Structured Streaming) | Systems requiring strict ACID transactions |

## 📘 About / Foundations
_Conceptual grounding for how the tool works._

### How It Works (Conceptual Model)

- Databricks runs on top of **Apache Spark**, providing distributed compute
- Data is stored in cloud object storage (S3/ADLS/GCS) with **Delta Lake** for ACID guarantees
- Workloads run on **clusters** (interactive or job clusters)
- **Unity Catalog** governs access, lineage, and metadata
- **Jobs** orchestrate pipelines, notebooks, and workflows
- **Repos** integrate Git for version control
- **Notebooks** allow SQL, Python, Scala, and R in a single environment

### System Contract

Databricks enforces contracts through:

- **Cluster configuration** (runtime version, autoscaling, node types)
- **Delta Lake schema enforcement**
- **Unity Catalog permissions**
- **Job definitions** (tasks, dependencies, triggers)
- **Workspace structure** (repos, notebooks, folders)

### Key Strategies

<Tabs>
<TabItem value="storage" label="Storage Strategy">

- Use **Delta Lake** for all production tables
- Enable schema evolution only when necessary
- Partition large datasets by natural keys (date, region, etc.)
- Responsibility: SWE + TA validation
- Goal: reliable, scalable, ACID‑compliant storage

</TabItem>

<TabItem value="compute" label="Compute Strategy">

- Use **job clusters** for scheduled workloads
- Use **autoscaling** to optimise cost
- Choose appropriate node types for workload patterns
- Responsibility: Platform + SWE
- Goal: efficient, cost‑aware compute usage

</TabItem>

<TabItem value="orchestration" label="Orchestration Strategy">

- Use Jobs for DAG‑based workflows
- Use task dependencies for multi‑step pipelines
- Integrate with CI/CD for deployment
- Responsibility: SWE + TA
- Goal: reliable, observable pipelines

</TabItem>
</Tabs>

```shell
Key Terminal Commands

# Run a job
databricks jobs run-now --job-id <id>

databricks jobs submit --json-file job.json

# Inspect Delta table metadata
spark.sql("DESCRIBE DETAIL my_table").show()

# View Delta table history
spark.sql("DESCRIBE HISTORY my_table").show()

# Read and write Delta tables
df = spark.read.format("delta").load("/mnt/data/table")
df.write.format("delta").mode("append").save("/mnt/data/table")

# Run a notebook programmatically
databricks runs submit --json-file run_notebook.json
```

## 🚨 Triage

:::danger
Symptoms and issues with this platform and how to triage this:
:::

| Symptom | TLDR of Root Cause | What to Check | Action to Propose |
|--------|---------------------|---------------|-------------------|
| Job failing | Notebook error, cluster misconfig, missing dependency | Job logs, cluster config | Fix code, adjust cluster, reinstall libs |
| Slow job performance | Underpowered cluster, skew, shuffle issues | Cluster metrics, Spark UI | Increase cluster size, repartition data |
| Delta table corruption | Interrupted writes, schema mismatch | `DESCRIBE HISTORY`, checkpoint logs | Restore version, fix schema, reprocess |
| Cluster not starting | Node type unavailable, network issue | Cluster events, VPC config | Change node type, fix networking |
| Streaming job stuck | Checkpoint corruption, late data | Streaming progress logs | Reset checkpoint, fix source data |
| Permission denied | Unity Catalog or workspace ACL issue | Permissions in UC | Update grants, fix workspace ACLs |
