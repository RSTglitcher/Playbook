:::tip Vault
**HashiCorp Vault** is a secrets management and data protection platform. It securely stores, distributes, and controls access to tokens, passwords, certificates, and encryption keys. Vault solves problems around secret sprawl, access auditing, and dynamic credential generation.
:::

---

## When to use

* When storing sensitive credentials, tokens, or keys centrally
* When dynamic secrets or short-lived credentials are needed
* When auditing and controlling access to secrets is required

**Do NOT use this tool when:**

* Secrets are static and access control is trivial
* Lightweight configuration files or environment variables suffice

---

## TA Skills Checklist

- Understand the secrets engines and authentication backends
- Validate access policies and permissions
- Audit vault usage and secrets consumption
- Monitor for expired or misconfigured secrets

<TabItem value="capabilities" label="TA Skills">

```shell
--------------
How to use Vault as a TA
--------------

# Login to Vault
vault login <token>        # Authenticate using a token
vault status               # Check Vault health and seal status
vault auth list            # See enabled authentication methods
vault secrets list         # List enabled secrets engines
vault read <path>          # Read a secret to verify access
vault policy list          # List all policies
vault audit list           # Check audit backends
vault token lookup         # Verify token metadata and TTL

# Monitor usage
vault kv list <path>       # List keys in KV secrets engine
vault read <path>          # Inspect secret value (read-only)
vault token revoke <token> # Revoke old or compromised tokens

# Validate setup
vault operator unseal -status  # Check unseal progress (if applicable)
```

</TabItem> <TabItem value="execution" label="SWE Skills">

--------------
Common ways engineers use Vault
--------------

```shell
# Dynamic database credentials
vault read database/creds/<role>  # Generates ephemeral credentials

# PKI certificates
vault write pki/issue/<role> common_name="example.com"

# Encrypt/decrypt data
vault write transit/encrypt/<key> plaintext=$(base64 <<< "sensitive data")
vault write transit/decrypt/<key> ciphertext=<ciphertext>

# Rotate secrets
vault lease revoke <lease_id>    # Revoke dynamic secrets

# CI/CD integration
export VAULT_TOKEN=<token>
vault kv get secret/<app>/<env> # Retrieve secrets in scripts
```

:::danger Failure Scenarios
Symptoms of issues with Vault and how to triage:
:::

| Symptom                     | TLDR of Root Cause                           | What to Check                          | Action to Propose                               |
|------------------------------|---------------------------------------------|---------------------------------------|------------------------------------------------|
| Vault sealed / unresponsive  | Vault not unsealed or service down          | `vault status`, pod/service logs      | Unseal Vault, restart service                  |
| Authentication fails         | Token expired, AppRole misconfigured        | Check token TTL, role policies        | Renew token, fix AppRole configuration        |
| Secrets not accessible       | Policy misconfigured or path disabled       | `vault policy read <name>`            | Adjust policies, enable required secrets engine |
| Dynamic secrets not generated| Database or external service misconfigured  | Engine configuration, lease status    | Fix secrets engine, regenerate credentials     |
| High latency or slow reads   | Vault overloaded or network issues          | Check server metrics, network health  | Scale Vault nodes, optimize queries           |
